Advice Generator - API Project

About project

This app was constructed to put into practice what I was learning about HTTP requests. I first wrote it with the XHR object before later rewriting it using the more up-to-date Fetch API. I learnt a lot about promises, updating the DOM with the parsed JSON information from the get request, chaining multiple .then() methods and error handling.

What I would improve...

Upon reviewing the project later I can see the code is a little verbose, I would put functions into the .then() / .catch() methods to handle the next steps as for both “search” and “random advice" the steps are almost the same. Moreover I have learnt more about using classes in Javascript (as seen in my Country Info App project), therefore I would go with this more sleek approach to writing code.
